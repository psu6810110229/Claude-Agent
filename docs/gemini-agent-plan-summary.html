Post-09 Plan

แผนเพิ่ม Gemini และ Multi-Provider Agent ให้ JARVIS

เป้าหมายคือเพิ่ม Gemini เป็นตัวเลือกของระบบ โดยยังรักษาแนวคิดเดิม: backend เป็นเจ้าของ state, AI เป็น proposal-only, และทุก write action ต้องผ่าน approval queue

สถานะตอนนี้

พร้อมเริ่ม

00-08 เสร็จแล้ว, manual test ผ่าน, และมี capability-contract.md เป็น safety contract

09 ไม่เป็น blocker

ใช้เป็น regression checklist ไม่ใช่ sprint implementation ที่ต้องรอก่อน

เริ่มแบบปลอดภัย

เริ่มจาก abstraction + stub ก่อน ยังไม่ใส่ secrets หรือยิง Gemini API จริงทันที

หลักการออกแบบ

  • Claude และ Gemini เป็น AI providers ไม่ใช่เจ้าของ state
  • Backend orchestrator เป็นเจ้าของ session, step, validation, guardrails, execution และ approval
  • Manual mode ต้องใช้ provider ที่ผู้ใช้เลือกเท่านั้น
  • Auto mode ต้องบอกผู้ใช้เสมอว่าเลือกหรือ fallback ไปใช้โมเดลไหน เพราะอะไร
  • ห้าม silent fallback และห้ามให้ AI execute write action โดยตรง

Roadmap สั้น ๆ

Phase 1

Provider Abstraction
ทำ interface กลางให้ Claude/Gemini คืน proposal schema เดียวกัน ใช้ stub test ก่อน

Phase 2

Manual Selection
ให้ผู้ใช้เลือก Claude หรือ Gemini ได้อย่างชัดเจน แต่ Gemini ยัง disabled ได้ถ้ายังไม่มี config

Phase 3

Gemini Provider
ต่อ Gemini API ผ่าน env config, ไม่ commit secrets, validate output ด้วย schema เดิม

Phase 4

Transparent Auto
เพิ่ม Auto selection พร้อม reason, trace, retry/fallback policy และ user-visible notice

Phase 5

Agent Steps
เพิ่ม AgentRun/AgentStep เพื่อรองรับงานหลายขั้น เช่น monthly P0 review, analysis, recommendation

Safeguards ที่ต้องมี

Permission Guard

แต่ละ step ต้องรู้ว่าทำได้แค่ read, analyze, propose หรือ approval-required

Step Budget

จำกัดจำนวน step, AI call, retry, runtime และ context size เพื่อกัน loop ไหล

Visible Trace

แสดง provider/model/reason เมื่อระบบเลือกหรือสลับโมเดล โดยเฉพาะใน Auto mode

สิ่งที่ยังไม่ทำตอนเริ่ม

  • ยังไม่เพิ่ม connector ใหม่ เช่น Gmail, Drive, Notion, filesystem scanning
  • ยังไม่เพิ่ม Google Calendar update/delete
  • ยังไม่ให้ Gemini หรือ Claude bypass approval queue
  • ยังไม่ทำ autonomous free-form agent loop ที่โมเดลสั่งต่อเองไม่จำกัด

คำตอบสั้นที่สุด

เราพร้อมเริ่ม Gemini แล้ว แต่ควรเริ่มจากฐานที่ไม่เสี่ยง: provider abstraction + manual selection + stubbed tests แล้วค่อยต่อ Gemini API และ Auto mode แบบโปร่งใสภายหลัง
